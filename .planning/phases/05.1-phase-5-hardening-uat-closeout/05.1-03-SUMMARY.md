## Deviations from Plan

### [Rule 3 — Auto-fix blocking issue] Integration-test `DependencyError` exception allowlist

- **Found during:** Task 2 GREEN verification.
- **Issue:** The integration test `test_init_then_closed_datagen_no_env_var` drives `_main_impl()` in-process on a dev box that may not have DLIO installed. When the HARDEN-03 fix correctly passes the env-var gate, the next thing `_main_impl()` does is check DLIO availability and raises `DependencyError` (NOT `_ShortCircuit` and NOT `SystemExit`). Without handling that exception, the GREEN test would still fail with an unhandled `DependencyError` — preventing the post-gate `assert called["capture_or_verify_invoked"]` from being reached.
- **Fix:** Added `from mlpstorage_py.errors import ConfigurationError, DependencyError` and an `except DependencyError: pass` branch to the in-process exception allowlist. Semantically: `DependencyError` means we got PAST the env-var gate (which is the only contract under test), and the downstream DLIO check has its own coverage in `test_benchmark_flow.py`.
- **Files modified:** `tests/integration/test_canonical_layout_end_to_end.py` (additions: import line + 6-line `except DependencyError` branch + an updated post-gate comment listing the four valid exit paths: `_ShortCircuit`, `SystemExit`, `DependencyError`, and "any non-ConfigurationError exception that occurs AFTER the helper has been invoked").
- **Commit:** Folded into GREEN (`3dbc2d3`) since the production fix and the test-side refinement are inseparable for confirming GREEN on the dev shell. Documented in the GREEN commit body under "Test integration refinement".
